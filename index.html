About LTR

Learning to Rank (LTR) is an open-source C++ library of algorithms for machine learning problems. LTR was initially designed for the problem of learning to rank but one can also use it for classification and regression problems. To the best of our knowledge there is no analogous open-source C++ libraries for learning to rank.

User can write code performing learning and testing by himself (see documentation) or can use a command-line application with human-readable configuration file.

Each trained ranker/classifier/regressor can be serialized in C++ code so it can be used in other projects with no dependecies on the LTR library.

Philosophy of LTR

From theoretical point of view every algorithm of machine learning can be decomposed to a few simple ideas.
Many of these ideas are similar or even the same for ranking, classification, regression.
Some examples of decomposition:

  • TreeNet = Gradient Descent + Boosting + Desicion Tree learning
  • SoftRank = Gradient Descent + Neural Net learning + Smoothing Nondifferentiability
  • RankPG = Genetic Programming + Listwise ranking measure optimizing
  • AdaBoost = Boosting + Exponential loss function approximation + Pointwise measure optimizing
  • AdaRank = Boosting + Exponential loss function approximation + Listwise measure optimizing

The main idea of LTR is to assist extensibility and generalizability of a source code. This is achieved by wide using of C++ templates and polymorphism.
One can easily add any heuristic to a standard learning algorithm because all of heuristics are functors in LTR. So user can easily change any step of algorithm by just defining new heuristic object and insert it into the code of the original algorithm.

User can write code that perform learning and testing by himself or can use CLI application with human-readable configuration file.

Each trained ranker/classifier/regressor can be serialized in C++ code so it can be used in other projects with no dependecies on the LTR library.

Requirements

  • Boost
    LTR assumes that you already have Boost sources and some static libraries in your system. The detailed inatallation manual is here.
  • CMake
    LTR uses the CMake build system so you should install it.

Compiler support

LTR is written following C++98 standard and it should be compatible with any common C++ compiler. LTR is being successfully built with the following compilers:

  • MSVS (Visual Studio) 2008 and newer
  • GCC 4.1 and newer
